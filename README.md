# MultiFarm — Stardew Valley Multiplayer Mod

Each player in a multiplayer session gets their own **private farm** (their choice of all 7 farm types), connected through a shared **Farm Hub**. Supports up to **8 players total** (world host + 7 guests).

## How it works

```
[Backwoods / Robin's]
         │
   [North Hub]   ← all 8 player portals
         │
[Farm] ─ [East Hub] ─ [BusStop]   ← all 8 player portals
         │
   [South Hub]   ← all 8 player portals
         │
  [Cindersap Forest]
```

Three shared **Farm Hubs** replace the vanilla connections:
- **East Hub** — between Farm and BusStop (main E-W route); connects north/south to the other hubs
- **North Hub** — between Backwoods and East Hub
- **South Hub** — between Cindersap Forest and East Hub

Each hub has portals for all 8 player farms, so players can reach any farm from any hub.

- The hubs replace the vanilla Farm ↔ BusStop warp and also connect north to Backwoods and south to Cindersap Forest
- When a new player joins, they pick a farm type from an in-game menu and are warped to their own private farm
- Each private farm has its own **farmhouse** (supports wallpaper, flooring, and furniture) and **mine cave**
- Players can only place buildings on **their own** farm — the carpenter menu is restricted per-player
- **Portal signs** — each portal in the hub displays the assigned player's name above it
- **Married NPCs follow you** — spouses re-route their daily schedule and home location to your private farmhouse automatically (via Harmony)
- All farms are persistent; returning players are warped straight to their farm

## Player installation

1. Install [SMAPI](https://smapi.io/) — follow the instructions on their site (one-time setup)
2. Download `MultiFarm-<version>.zip` from the releases page
3. Extract the zip into your `Stardew Valley/Mods/` folder
4. Launch the game through SMAPI

That's it. No other prerequisites.

## Building from source

### Prerequisites

- [.NET 6 SDK](https://dotnet.microsoft.com/en-us/download/dotnet/6.0)
- [SMAPI](https://smapi.io/) installed into your Stardew Valley folder
- Stardew Valley (Steam or GOG)

### Build

```bash
cd MultiFarm
dotnet build                  # debug build, deploys to your Mods folder automatically
dotnet build -c Release       # release build + produces MultiFarm-<version>.zip
```

The `Pathoschild.Stardew.ModBuildConfig` NuGet package auto-detects your Stardew Valley install on Windows, macOS, and Linux — no environment variables needed.

The distributable zip is written to `MultiFarm/bin/Release/net6.0/MultiFarm-<version>.zip`.

### Regenerating maps

Map TMX files are generated by `tools/gen_maps.py`. You need Python 3 and the vanilla Stardew Valley TMX files on your machine.

```bash
cd tools
python gen_maps.py
# Writes to ../MultiFarm/assets/maps/
```

## Assets

All maps live in `MultiFarm/assets/maps/`:

| File | Purpose |
|------|---------|
| `MultiFarm_Hub_East.tmx` | East Hub — between Farm (west) and BusStop (east); N/S exits to North/South hubs |
| `MultiFarm_Hub_North.tmx` | North Hub — between Backwoods (north) and East Hub (south) |
| `MultiFarm_Hub_South.tmx` | South Hub — between East Hub (north) and Forest (south) |
| `PlayerFarm_0.tmx` … `PlayerFarm_6.tmx` | Per-type private farms (one per player) |
| `PlayerFarmCave.tmx` | Shared cave template (instantiated per player) |
| `PlayerFarmHouse.tmx` | Shared farmhouse template (instantiated per player, `DecoratableLocation`) |

Hub maps are generated by `tools/gen_maps.py`. Use [Tiled](https://www.mapeditor.org/) to edit `.tmx` files manually. Warp tile positions must match the entry point constants in `FarmHubManager.cs`.

## Config (`config.json`)

| Key | Default | Description |
|-----|---------|-------------|
| `MaxPlayers` | `8` | Number of farm slots (1–8) |
| `AllowedFarmTypes` | `[0,1,2,3,4,5,6]` | Farm types players can choose from |
| `ReplaceVanillaWarps` | `true` | Route Farm ↔ BusStop / Backwoods / Forest through hub |

## Console commands (in-game SMAPI console)

| Command | Description |
|---------|-------------|
| `mf_status` | Show all slot assignments and farm types |
| `mf_assign <name> [farmType]` | Manually assign a player to the next open slot (farmType 0–6, optional) |
| `mf_goto <slot>` | Warp yourself to a farm slot |
| `mf_selectfarm` | Re-open the farm selection menu for the local player |

## Multiplayer notes

- The **host** (main player) manages all slot assignments and syncs state to clients on join
- Clients receive their assignment via SMAPI mod messages; no direct file I/O from clients
- Player identity uses `UniqueMultiplayerID` (stable across sessions), not display name
- Starter items (tools + seeds) are granted exactly once per player per save
- Maximum **8 players total**: world host counts as one of the 8 slots (host + 7 guests)
